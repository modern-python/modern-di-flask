# Change: Open the per-request child container (modern-di 3.x mandatory-open)

**Lane:** lightweight — 1 line of adapter source, 3 test files, no new file,
no public-API change, existing tests as the regression suite.

## Goal

modern-di 3.0 made container opening mandatory: a freshly-built container
(root or child) must be opened via `.open()`/`with`/`async with` before
`resolve`/`build_child_container` — otherwise `ContainerClosedError`. Bumping
the `modern-di` floor to `>=3,<4` broke `modern-di-flask` two ways:

1. **Adapter bug:** `_enter_request` builds the per-request `REQUEST` child
   and stashes it on `flask.g`, but building and closing span two separate
   Flask hooks (`before_request` / `teardown_appcontext`) — there is no
   enclosing `with` block to open it implicitly, so the child was never
   opened. Every resolve against it raised `ContainerClosedError`.
2. **Root container:** the maintainer ruled that `setup_di` must NOT open
   the root container — Flask has no application-startup hook, so root
   lifecycle (open and close alike) is the caller's own responsibility,
   symmetric with the pre-existing "teardown is yours to own" contract.
   Only the test suite's own container construction needed to change to
   exercise this real path.

## Approach

- `modern_di_flask/main.py::_enter_request` — call `child.open()`
  immediately after `build_child_container`, before stashing it on `g`. This
  is the actual bug fix (adapter source).
- `setup_di` is unchanged — it still does not open the root container, per
  the maintainer's ruling.
- `tests/conftest.py::container` fixture now opens the root container it
  builds (`with Container(...) as container_: yield container_`), and the two
  inline `Container(...)` constructions in `tests/test_middleware.py` do the
  same, so the suite exercises the real caller-owned-open path instead of an
  artifact of the old always-open 2.x default.
- README and `architecture/dependency-injection.md` updated: the root
  container's lifecycle (open **and** close) is caller-owned; passing an
  unopened container to `setup_di` now fails on the first request.

See [architecture/dependency-injection.md](../../architecture/dependency-injection.md)
for the promoted capability description.

## Files

- `modern_di_flask/main.py` — `_enter_request` opens the child it builds.
- `tests/conftest.py` — `container` fixture opens the root container via `with`.
- `tests/test_middleware.py` — the two inline `Container(...)` sites open
  their root container via `with`.
- `README.md` — usage example opens the root container before `setup_di`;
  the caller-owned-teardown sentence extended to cover open.
- `architecture/dependency-injection.md` — "No root lifecycle wiring" and
  "Per-request scope" sections updated to describe the mandatory-open reality.
